## BUG-017 修复：SplendorBeliefTracker 偷看牌堆内容

`randomize_unseen` 直接从 `data.decks` 读取真实牌堆内容构建 unseen pool——等于 AI 精确知道牌堆组成。

修复方案：
- 新增 `seen_cards_: std::unordered_set<int>` 成员，仅通过 `init` 和 `observe_action` 维护
- `init`：首次调用扫描 tableau + visible reserved + 自己的 reserved
- `observe_action`：BuyFaceup/ReserveFaceup 时比较 tableau 差异找新翻卡；ReserveDeck 时如果是自己执行则记录暗牌
- `randomize_unseen`：用 `全卡池(90) - seen_cards_` 按 tier 构建 unseen pool，不再读 `data.decks`

关键设计原则已记入 CLAUDE.md：belief tracker 永远不能读取游戏状态的隐藏字段。

## N 维 Value Head 支持多人游戏

### BUG-018 修复：三个多人游戏 bug

1. Adjudicator z_values 用 `+1/-1` 而非零和的 `+1/-1/(n-1)`
2. ONNX evaluator 标量分支展开 `[v, -v, -v]` 而非 `[v, -v/(n-1), -v/(n-1)]`
3. N 维分支缺少 perspective→absolute 旋转

### N 维 value head 实现

Value head 从 `nn.Linear(prev, 1)` 改为 `nn.Linear(prev, num_players)`：

- **perspective-relative 输出**：与 encoder 旋转对齐，`values[0]` = 我的 value
- **MCTS backup**：evaluator 旋转回绝对顺序，`leaf_values[to_play]` 逻辑不变
- **训练 target**：`rotate_z_values(z_values, player)` 替代标量 `z`
- **零和**：不在网络架构强制，由 `IStateValueModel` 的 target 保证，selfplay 加 debug assert 验证
- **统一 N 维**：2 人也输出 2 维，不特殊处理

### game_metadata() pybind 函数

新增 `dinoboard_engine.game_metadata(game_id)` 返回 `{num_players, action_space, feature_dim}`。解决变体（如 `splendor_3p`）无法从 JSON 获取正确维度的问题。CLI 和测试框架统一用此函数获取 C++ 侧的真实维度。

- `training/pipeline.py`：`rotate_z_values()` + N 维 loss

## 单搜索分析（Single-Search Analysis）

将动作掉分分析从"双搜索"改为"单搜索"方案。

**旧方案**：precompute MCTS 获取最优胜率 → 用户走棋后再做一次 probe MCTS（对用户选择的动作强制走一步后搜索）获取实际胜率。两次搜索结果不在同一棵树上，且 probe 的 value 取反在 3p+ 下不成立。

**新方案**：MCTS `root_edge_values` 记录每个根边的 N 维 per-player Q 均值。C++ 在 backup 时累积所有叶子的 per-player value，搜索结束后导出为 `stats.root_edge_values`。pybind 导出为 `stats["action_values"]`：`{action_id: [p0_q, p1_q, ...]}`。

分析逻辑：
- `best_wr = (root_values[human_player] + 1) / 2`
- `actual_wr = (action_values[chosen_action][human_player] + 1) / 2`
- 如果用户选了最优动作：`actual_wr = best_wr`

优势：一次搜索覆盖所有动作；N 维 Q 值直接适用于任意人数游戏；去掉 probe 后分析延迟降低约 50%。

文件改动：
- `engine/search/net_mcts.cpp/h`：`root_edge_values` 累积和导出
- `bindings/py_engine.cpp`：导出 `action_values` dict
- `platform/game_service/pipeline.py`：去掉 probe，`_human_wr_from_stats` / `_human_wr_for_action` 从 action_values 提取
